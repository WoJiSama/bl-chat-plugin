## 1. 规则包是什么

规则包是一份 YAML 文件，用来声明：

- 一套规则的名称、别名和说明；
- 每位用户在这套规则下拥有哪些人物卡字段；
- 这些字段如何互相计算，例如 `effective_str = strength - injury`；
- 用户可以输入哪些骰娘命令、命令带哪些参数；
- 命令要掷哪些骰、如何判断结果、命中后如何改人物卡；
- 最终如何组织回复文本。

YAML 是数据，不是程序。规则包不能包含 JavaScript、shell 命令、网络 URL、文件路径或函数体。所有公式只会交给骰娘的受限表达式解析器执行。

这条边界很重要：它既允许 `a = b - c`、成功等级、伤害、资源扣除和随机表，也避免一个他人分享的 YAML 获得服务器权限。

### 1.1 内置规则与自定义规则的关系

COC、DND 等已经实现的复杂规则继续由内置模块负责。自定义规则包不覆盖它们的内部实现，而是新增一套可声明的规则。

- 内置 `.ra`、`.r`、`.sc`、`.st` 等命令始终优先；
- 自定义规则默认通过“规则包前缀 + 子命令”使用，例如 `.fate check 3 4`；
- 只有管理员在当前群显式启用规则包且前缀没有冲突时，包 ID 和别名才会生效；
- 自定义规则包的数据与 COC 人物卡隔离，不会覆盖 `STR`、`SAN` 等已有数据。

## 2. 五分钟：导入第一个规则包

以下命令是当前实际交互约定。命令开头的 `.` 也可写成中文句号 `。`，命令字与参数之间可有或没有空格；参数内部仍按对应命令的固定顺序解析。

1. 下载或编写 YAML，例如 [`examples/fate-lite.yaml`](examples/fate-lite.yaml)。
2. 在群里发送 YAML 文件或 YAML 代码块，然后引用该消息发送 `.骰规则导入`；也可以把 YAML 代码块直接放在导入命令后。
3. 骰娘只做预检，不立即启用。预检结果会列出规则包 ID、包版本、命令、字段和试掷结果；当前群与其他活动包的前缀冲突会在启用时检查。
4. 主人确认后输入 `.骰规则确认 fate-lite`。
5. 在需要的群输入 `.骰规则启用 fate-lite`。
6. 群成员使用 `.fate check 3 4`。

常用管理命令：

| 命令 | 作用 | 权限 |
| --- | --- | --- |
| `.骰规则导入` | 从被引用的 YAML 文件或代码块创建待确认版本 | 主人 |
| `.骰规则预览 <id>` | 查看待确认或当前版本的校验报告 | 主人、群管理员 |
| `.骰规则确认 <id>` | 将预检通过的版本写入规则库 | 主人 |
| `.骰规则列表` | 列出已导入规则包、版本和启用群数 | 主人、群管理员 |
| `.骰规则查看 <id[@版本]>` | 查看元数据、命令、字段与版本 | 主人、群管理员 |
| `.骰规则启用 <id[@版本]>` | 在当前群启用一个已确认规则包；省略版本时用最新版本 | 群管理员或主人 |
| `.骰规则禁用 <id>` | 在当前群禁用规则包，不删除数据 | 群管理员或主人 |
| `.骰规则导出 <id[@版本]>` | 发送原始 YAML 文件；省略版本时导出最新版本 | 主人、群管理员 |
| `.骰规则回滚 <id> <版本>` | 将某群的活动版本切回已保存版本 | 主人 |
| `.骰规则删除 <id> 确认` | 从所有群禁用并归档规则文件，保留人物卡数据 | 主人 |
| `.骰规则恢复 <id>` | 从最新有效归档恢复规则包；恢复后不自动启用 | 主人 |
| `.骰规则帮助` | 查看管理命令摘要 | 所有人 |

普通成员不能导入、覆盖或删除全局规则包。每个群只选择“启用哪些包及哪个版本”；用户人物卡不会因为群里禁用规则包而被删除。

## 3. 完整文件结构

所有规则包最外层字段如下。未知字段默认是校验错误，避免拼写错后被静默忽略。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `version` | 整数 | 是 | 规则包格式版本。V1 固定为 `1`。 |
| `id` | 标识符 | 是 | 全局唯一稳定 ID，只能用小写字母、数字、`-`，长度 3-48。不能在更新时修改。 |
| `name` | 字符串 | 是 | 面向用户展示的规则名，最长 60 字。 |
| `aliases` | 字符串数组 | 是 | 调用规则包时使用的前缀，例如 `fate`、`命运`。 |
| `description` | 字符串 | 否 | 简短介绍，最长 300 字。 |
| `compatibility` | 对象 | 否 | 声明规则包自身的版本、迁移和需要的运行时能力。 |
| `identity` | 对象 | 否 | 骰娘回复中如何显示角色名称。 |
| `character` | 对象 | 否 | 人物卡字段、默认值、派生值和资源。 |
| `group` | 对象 | 否 | 当前群、当前规则包共享的字段。运行时 2.0。 |
| `statuses` | 对象 | 否 | 持续状态、层数、持续时间和生命周期动作。运行时 2.0。 |
| `items` | 对象 | 否 | 物品数量、堆叠上限和装备槽。运行时 2.0。 |
| `abilities` | 对象 | 否 | 技能/法术、等级、冷却和资源消耗。运行时 2.0。 |
| `events` | 对象 | 否 | 团务、轮次与行动开始/结束时执行的声明式动作。运行时 2.0。 |
| `dice_sets` | 对象 | 否 | 非标准骰定义，例如 Fate 骰和符号骰。 |
| `tables` | 对象 | 否 | 可抽取的固定随机表。 |
| `commands` | 数组 | 是 | 规则包暴露的命令。至少一条，最多 30 条。 |
| `templates` | 对象 | 否 | 可复用输出模板。 |
| `limits` | 对象 | 否 | 在系统硬上限以内进一步收紧本包限制。 |

最小可运行规则包：

```yaml
version: 1
id: simple-d6
name: 简单 D6
aliases: [d6]
commands:
  - id: roll
    aliases: [roll, 掷骰]
    rolls:
      main: "1d6"
    output: "{actor} 掷出了 {roll.main.total}。"
```

用户输入 `.d6 roll` 或 `.d6 掷骰` 即可调用。`id` 是机器用稳定键；`name`、`aliases`、`label` 才是给人看的文字。

### 3.1 标识符规则

字段 ID、命令 ID、骰点名、随机表名和属性 ID 使用同一规则：`[a-z][a-z0-9_]{0,47}`。

- 可展示中文，但不要把中文作为数据 ID；例如 `id: stealth`、`label: 潜行`。
- ID 一经发布就不要改。改展示名或别名不会丢用户数据，改 `id` 会被视为新字段或新规则包。
- ID 总是通过命名空间访问，因此参数可以叫 `target`，字段可以叫 `status`：分别写成 `arg.target`、`attr.status`。不要把顶层命名空间名称本身当作“全局变量”使用；表达式中始终写完整前缀。

### 3.2 `compatibility`

```yaml
compatibility:
  min_runtime: "1.0"
  package_version: "1.2.0"
  migrations:
    - from: "1.0.0"
      rename_fields:
        old_hp: hp
```

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `min_runtime` | 字符串 | 最低骰娘规则运行时版本；不满足时不能启用。 |
| `package_version` | 字符串 | 规则作者自己的版本号，用于展示和匹配数据迁移来源；仓库回滚使用导入时生成的整数版本。 |
| `migrations` | 数组 | 受限的数据迁移声明。V1 只支持字段重命名与新增默认值。 |

导入预检会验证迁移字段、目标类型和默认值。确认新版本不会扫描或批量改写所有人物卡；用户下一次使用该规则包时，运行时根据人物卡记录的 `package_version` 惰性应用一条 `from` 精确匹配的迁移。V1 迁移不能运行任意公式，不允许链式重命名或多个来源写入同一目标，也不提供删除字段动作。

每个 `migrations` 项支持：

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `from` | 字符串 | 来源 `package_version`，必填。 |
| `rename_fields` | 对象 | 旧字段 ID 到新字段 ID 的映射。目标必须已存在。 |
| `add_defaults` | 对象 | 新增字段的默认值；只为没有该字段的旧人物卡写入。 |

### 3.3 `templates` 与 `limits`

`templates` 的每个键都是标识符，值是输出模板字符串：

```yaml
templates:
  short_result: "{actor}：{result}。"
  full_result: "{actor} 掷出 {roll.check.detail} = {roll.check.total}，{result}。"
```

`limits` 只能比系统默认限制更严格，不能借规则包放大服务器允许的负载：

| 字段 | 类型 | 作用 |
| --- | --- | --- |
| `max_dice_count` | 正整数 | 本规则包一次命令允许掷出的骰子总数。必须不大于骰娘全局 `maxDiceCount`。 |
| `max_rolls` | 正整数 | 单条命令允许的命名骰点数。必须不大于系统上限 12。 |
| `max_expression_length` | 正整数 | 本规则包每条表达式的最大字符数。必须不大于系统上限 512。 |
| `max_output_length` | 正整数 | 本规则包单条输出上限。必须不大于系统上限 2,000。 |
| `max_table_entries` | 正整数 | 每张随机表的最大条目数。必须不大于系统上限 100。 |

## 4. 角色、人物卡与显示名称

一个 QQ 用户可以拥有多张人物卡，和现有 `.pc new/use` 的使用习惯一致。规则包的数据存放在该人物卡的 `ruleData[规则包 id]` 下，不和 COC、DND 数据混写。

### 4.1 `character.fields`

```yaml
character:
  fields:
    name:
      type: string
      label: 角色名
      default: "{sender.card}"
      max_length: 30
    strength:
      type: integer
      label: 力量
      default: 50
      min: 0
      max: 100
    injury:
      type: integer
      label: 伤势
      default: 0
      min: 0
    effective_strength:
      type: integer
      label: 有效力量
      formula: "attr.strength - attr.injury"
```

每个字段支持的参数：

| 参数 | 适用类型 | 作用 |
| --- | --- | --- |
| `type` | 全部 | `integer`、`number`、`string`、`boolean` 之一。必填。 |
| `label` | 全部 | 卡面和错误提示显示的名称；默认使用字段 ID。 |
| `description` | 全部 | 字段说明，最长 200 字。 |
| `default` | 全部 | 创建规则数据时的默认值。字符串允许 `{sender.card}`、`{sender.nickname}`。 |
| `formula` | 数值型 | 派生字段公式；定义后字段只读，不能与 `default` 同时使用。 |
| `min` / `max` | 数值型 | 写入值的闭区间限制。 |
| `step` | 数值型 | 编辑提示步长，不改变计算精度。 |
| `enum` | 字符串、整数 | 可选值数组；写入其他值会失败。 |
| `max_length` | 字符串 | 最大字符数，范围 1-200。 |
| `secret` | 全部 | 为 `true` 时只有 GM、群管理员和主人可查看或修改。引用该字段或依赖它的派生字段的自定义命令必须声明 `permission: gm` 或更高；显示名称禁止引用秘密字段。 |
| `persistent` | 非公式字段 | 默认为 `true`。设为 `false` 时每条规则命令都从默认值重新开始，只能由该命令的 action 临时修改，不写入人物卡，也不能用通用 `设/删` 修改。 |

`formula` 用于“始终跟随其他属性变化的值”。例如 `effective_strength = strength - injury`。它不是“执行一次并保存”的赋值；想保存快照，请在命令 `actions` 中使用 `set`。

### 4.2 显示名称

```yaml
identity:
  display_name: "{attr.name}（{attr.class}）"
  fallback: "{sender.card}"
```

| 字段 | 作用 |
| --- | --- |
| `display_name` | 回复中 `{actor}` 的模板。可引用字符串字段和派生数值。 |
| `fallback` | 模板计算为空或字段未创建时使用；默认 `{sender.card}`。 |

这只影响骰娘的回复显示，例如“艾琳（游侠）进行潜行检定”。它**不会**修改 QQ 群名片、昵称或账号资料。

### 4.3 用户如何改角色数据

规则包会自动提供受字段 schema 约束的通用命令：

```text
.fate 卡                    # 查看本规则包当前人物卡
.fate 设 name=艾琳 skill=45 # 写入字段
.fate 查 effective_skill    # 查询一个字段（会计算公式）
.fate 删 temporary_flag     # 删除可删除的可持久字段值，回到默认值
```

规则作者也可在 `commands` 里定义更自然的专用录卡命令。通用命令不允许改写 `formula` 或 `persistent: false` 字段；普通玩家也不能通过通用命令读取、设置或删除 `secret` 字段。

## 5. 命令、参数和骰点

每项 `commands` 描述一个子命令。

```yaml
commands:
  - id: stealth
    aliases: [stealth, 潜行]
    label: 潜行检定
    description: 用有效敏捷进行潜行检定。
    arguments:
      - id: difficulty
        label: 难度
        type: integer
        required: true
        min: 0
        max: 100
      - id: bonus
        label: 加值
        type: integer
        default: 0
    rolls:
      check: "1d100 + arg.bonus"
    output: "{actor} 潜行：{roll.check.total}。"
```

### 5.1 命令参数

| 参数 | 类型 | 必填 | 作用 |
| --- | --- | --- | --- |
| `id` | 标识符 | 是 | 在公式里以 `arg.<id>` 引用。 |
| `aliases` | 字符串数组 | 是 | 此子命令可接受的文字。 |
| `label` | 字符串 | 否 | 帮助、错误提示中的名称。 |
| `description` | 字符串 | 否 | 帮助文本。 |
| `permission` | 枚举 | 否 | `player`、`gm`、`admin`、`master`；默认 `player`。 |
| `visibility` | 枚举 | 否 | `public`、`private`、`gm`；默认 `public`。 |
| `public_output` | 字符串 | 否 | 私密结果在群内显示的占位模板。 |
| `arguments` | 数组 | 否 | 参数定义，按声明顺序解析。 |
| `rolls` | 对象 | 否 | 命名骰点；键名会成为 `roll.<名称>`。 |
| `draws` | 对象 | 否 | 命名抽表；键名会成为 `table.<名称>`，值是顶层 `tables` 的表 ID。 |
| `let` | 对象 | 否 | 只读中间计算值；在 `rolls` 之后计算。 |
| `opposed` | 对象 | 否 | 声明双方数值、胜负方向和平局策略。 |
| `branches` | 数组 | 否 | 自上而下命中第一个条件分支。 |
| `actions` | 数组 | 否 | 无论分支如何，都要执行的状态变更。 |
| `template` | 标识符 | 否 | 引用顶层 `templates` 中的模板；不能与 `output` 同时使用。 |
| `output` | 字符串 | 否 | 默认输出模板。 |
| `error_output` | 字符串 | 否 | 执行错误时的用户提示模板，只能引用 `{message}`，不暴露内部堆栈或未完成的执行上下文。 |

一个参数项支持：

| 字段 | 说明 |
| --- | --- |
| `id` | 参数 ID，必填。 |
| `type` | `integer`、`number`、`string`、`boolean`、`enum`、`actor`。必填。 |
| `required` | 是否必须提供；默认 `false`。 |
| `default` | 未提供时的值。`required: true` 时不得设置。 |
| `min` / `max` | 数字范围。 |
| `enum` | 枚举候选，`type: enum` 时必填。 |
| `rest` | `true` 时吸收剩余文本；一个命令最多一个，且必须是最后一个参数。 |
| `allowed` | 仅 `actor`。允许的目标类型数组：`self`、`member`、`npc`。默认仅 `self`。 |
| `multiple` | 仅 `actor`。为 `true` 时接收多个固定目标选择器，并且必须同时设置 `rest: true`。 |
| `description` | 参数说明。 |

例如 `.shadow 潜行 20 +5` 会按顺序得到 `arg.difficulty = 20`、`arg.bonus = 5`。带空格的名称使用 `rest: true`，不要依赖模糊猜测。

### 5.2 `rolls` 的执行规则

```yaml
rolls:
  check: "1d100"
  damage: "2d6 + derived.damage_bonus"
```

- 每个键是命名骰点，最多 12 个；
- 从上到下执行；后一个骰点可读取前一个，例如 `bonus_damage: "roll.damage.total / 2"`；
- 每个骰点在一次命令中只执行一次；重复引用 `roll.damage.total` 不会重新掷骰；
- 结果包含 `total`、`detail`、`expr`，并可在模板中使用；
- 骰点表达式必须返回数值，字符串和布尔值只能在 `let`、条件或输出中使用。

## 6. 值引用、公式和复杂运算

### 6.1 命名空间

为了避免“力量”到底是参数还是人物卡字段的歧义，所有引用都使用前缀。

| 写法 | 含义 | 示例 |
| --- | --- | --- |
| `arg.<id>` | 当前命令参数 | `arg.difficulty` |
| `attr.<id>` | 可持久人物卡字段 | `attr.hp` |
| `derived.<id>` | 派生字段 | `derived.effective_strength` |
| `roll.<id>.total` | 已执行骰点总值 | `roll.attack.total` |
| `roll.<id>.detail` | 已执行骰点明细，只能输出 | `{roll.attack.detail}` |
| `let.<id>` | 当前命令中间计算值 | `let.margin` |
| `table.<id>` | 已抽取的随机表项 | `table.critical.text` |
| `result` | 当前分支设置的结果 | `result` |
| `actor` | 当前显示名称，只能输出 | `{actor}` |
| `sender.card` | 当前 QQ 群名片 | `{sender.card}` |
| `group.id` | 当前群 ID，只能用于相等比较和日志分区 | `group.id == 123` |
| `target.*` | 当前目标角色摘要 | `target.attr.hp`、`target.name` |
| `shared.<id>` | 当前群共享字段 | `shared.threat` |
| `session.*` | 当前团务状态 | `session.round`、`session.phase` |
| `inventory.<id>.*` | 当前角色物品状态 | `inventory.sword.quantity`、`inventory.sword.equipped` |
| `abilities.<id>.*` | 当前角色能力状态 | `abilities.fireball.rank`、`abilities.fireball.cooldown` |
| `opposed.*` | 对抗结果 | `opposed.actor`、`opposed.target`、`opposed.winner` |
| `status.*` | 状态生命周期动作中的当前状态 | `status.stacks`、`status.duration` |

`attr` 默认指命令发起者的当前人物卡。只有规则包显式声明 `actor` 参数，并在 action 中写明 `scope: target` 和对应参数 ID，才可访问或修改目标草稿；运行时还会再次检查目标类型和命令权限。表达式仍不能访问任意 QQ 资料、群文件或聊天记录。

### 6.2 数值表达式语法

支持：

```text
1 + 2 * 3
(attr.strength - attr.injury) / 5
roll.damage.total + max(0, attr.strength - 50)
floor(attr.hp / 2)
clamp(attr.san - roll.loss.total, 0, attr.max_san)
```

运算符优先级从高到低：

| 优先级 | 运算符 | 说明 |
| --- | --- | --- |
| 1 | `()` | 括号。 |
| 2 | 一元 `+`、`-`、`!` | 正、负、逻辑非。 |
| 3 | `*`、`/`、`%` | 乘、除、取余。除零为执行错误。 |
| 4 | `+`、`-` | 加、减；两个字符串可用 `+` 拼接。 |
| 5 | `<`、`<=`、`>`、`>=` | 大小比较。 |
| 6 | `==`、`!=` | 严格相等/不等，不做字符串转数字猜测。 |
| 7 | `&&` | 逻辑与，短路。 |
| 8 | `||` | 逻辑或，短路。 |
| 9 | `?:` | 三元条件，例如 `attr.hp > 0 ? "存活" : "倒下"`。 |

支持的函数：

| 函数 | 说明 |
| --- | --- |
| `min(a, b, ...)` / `max(a, b, ...)` | 最小值/最大值。 |
| `clamp(value, min, max)` | 限制在区间内。 |
| `abs(value)` | 绝对值。 |
| `floor(value)` / `ceil(value)` / `round(value)` | 向下、向上、四舍五入。 |
| `sqrt(value)` | 平方根，负数报错。 |
| `pow(base, exponent)` | 幂；指数与结果受数值上限限制。 |
| `if(condition, yes, no)` | 条件表达式。 |
| `coalesce(a, b)` | `a` 为 `null` 时取 `b`。 |
| `len(text)` | 字符串长度。 |

不支持变量赋值、循环、递归、匿名函数、对象字面量、数组运算、正则、`eval`、`Date`、随机 API 或外部调用。随机性只能通过 `rolls`、`dice_sets` 和 `tables` 产生，保证一次命令可复盘。

### 6.3 派生字段依赖与循环

以下合法：

```yaml
fields:
  strength: { type: integer, default: 50 }
  injury: { type: integer, default: 0 }
  effective_strength:
    type: integer
    formula: "attr.strength - attr.injury"
  damage_bonus:
    type: integer
    formula: "floor(derived.effective_strength / 10)"
```

以下非法，会在导入阶段报“循环依赖”：

```yaml
fields:
  a: { type: integer, formula: "derived.b + 1" }
  b: { type: integer, formula: "derived.a + 1" }
```

系统会把派生字段构造成依赖图，按拓扑顺序计算。未知引用和循环依赖会在导入时拒绝；具体数据造成的类型错误、除零、越界和超限会在无副作用试掷或真实执行时拒绝。

## 7. 条件、分支、结果表和状态更新

### 7.1 `branches`

`branches` 自上而下判断，第一个 `when` 为真的分支获胜；最后建议写一个没有 `when` 的默认分支。

```yaml
branches:
  - when: "roll.check.total <= 5"
    result: 大成功
    output: "{actor} 灵光一现：{roll.check.total}，大成功！"
  - when: "roll.check.total <= derived.effective_skill"
    result: 成功
  - result: 失败
```

分支字段：

| 字段 | 作用 |
| --- | --- |
| `when` | 布尔表达式；省略表示默认分支。 |
| `result` | 写入当前 `result` 的字符串，供后续 action 与模板引用。 |
| `let` | 只在该分支内生效的中间值。 |
| `actions` | 只有命中本分支才执行的状态更新。 |
| `output` | 覆盖命令默认输出。 |
| `tags` | 结构化结果标签，例如 `[success, critical]`，供日志和未来卡面使用。 |

### 7.2 `actions`

动作默认操作发起者人物卡的非派生字段。运行时 2.0 可通过受限 `scope` 操作已声明目标或群共享字段。动作按顺序计算，但整次命令全部校验成功后才一次提交；中间失败不会留下半次扣血。

```yaml
actions:
  - op: add
    field: hp
    value: "-roll.damage.total"
  - op: clamp
    field: hp
    min: 0
    max: "attr.max_hp"
  - op: set
    field: status
    value: "attr.hp <= 0 ? '倒下' : '受伤'"
```

| `op` | 必填字段 | 作用 |
| --- | --- | --- |
| `set` | `field`、`value` | 将字段设为表达式结果。 |
| `add` | `field`、`value` | 数值相加；可用负值扣除。 |
| `subtract` | `field`、`value` | 数值相减。 |
| `min` | `field`、`value` | 字段与值取较小者。 |
| `max` | `field`、`value` | 字段与值取较大者。 |
| `clamp` | `field`、`min`、`max` | 把字段限制在区间内。 |
| `clear` | `field` | 删除当前值，下一次读取回退到默认值。 |
| `add_status` / `remove_status` | `status` | 添加或移除已声明状态；添加可带 `duration`、`stacks`。 |
| `add_item` / `remove_item` | `item` | 增减物品；可带 `quantity`。 |
| `equip` / `unequip` | `item` | 装备或卸下物品；同一装备槽互斥。 |
| `learn_ability` / `forget_ability` | `ability` | 学习或遗忘技能/法术；学习可带 `rank`。 |
| `use_ability` / `reset_ability` | `ability` | 使用能力或清空冷却。 |

动作的 `scope`：

| `scope` | 目标 | 额外要求 |
| --- | --- | --- |
| `actor` | 命令发起者或当前生命周期角色 | 默认值。 |
| `target` | 已解析的角色目标 | 必须设置 `target: <actor参数ID>`。 |
| `group` | 当前群的 `group.fields` | 只支持普通字段动作，不支持角色状态、物品和能力。 |

每个动作可选 `when` 字段，条件不满足就跳过。例如：

```yaml
- op: add
  field: luck
  value: 1
  when: "result == '大成功'"
```

不能对 `formula` 字段执行动作。跨角色和群写入只能使用已声明 scope，不能用动态路径绕过 schema；访问 `secret` 字段的命令至少需要 `permission: gm`。

### 7.3 执行顺序

一次命令严格按下面顺序执行：

1. 校验并解析命令参数；
2. 读取发起者、已声明目标和群共享状态的事务草稿，补默认值并计算派生字段；
3. 从上到下执行 `rolls`；
4. 执行 `draws`，把每张随机表的唯一抽取结果写入 `table`；
5. 计算命令级 `let`；
6. 命中一个 `branches` 分支；
7. 执行命令级与分支级 `actions`；
8. 校验所有写入结果的类型、范围和枚举；
9. 解析私密接收者并渲染输出；如果没有合格接收者则终止；
10. 在同一状态事务中写入所有角色、群状态和结构化审计；
11. 记录普通团录，并按可见性把正文交给群或私聊发送层。

同一动作列表里后续动作会读取前一动作的暂存结果；输出默认读取写入后的新值。需要旧值时，在 `let` 里先保存，例如 `old_hp: "attr.hp"`。

## 8. 模板和输出

输出模板可以直接写在 `output`，或统一写入顶层 `templates` 后用 `template: 名称` 引用。

```yaml
templates:
  normal_check: "{actor} 进行 {command.label}：{roll.check.detail} = {roll.check.total}，结果：{result}。"

commands:
  - id: check
    aliases: [check, 检定]
    rolls: { check: "1d100" }
    template: normal_check
```

模板占位符采用 `{路径}`，支持：

```text
{actor}
{command.label}
{arg.difficulty}
{attr.hp}
{derived.effective_strength}
{roll.check.expr}
{roll.check.detail}
{roll.check.total}
{let.margin}
{table.critical.text}
{result}
```

模板不会执行表达式。请在 `let`、`formula` 或 `output` 前先计算好值。不存在的占位符会在导入预检时报错，而不是悄悄显示空白。

输出最多 2,000 个字符；超过时执行失败并提示规则作者缩短模板。规则包中的 CQ 消息码会在导入时拒绝，运行时输出还会再次清理控制字符和 CQ 前缀，不会把 YAML 文本当成管理动作执行。

## 9. 非标准骰、随机表与资源

### 9.1 `dice_sets`

非标准骰定义骰面，不定义代码。

```yaml
dice_sets:
  fate:
    label: Fate 骰
    faces: [-1, -1, 0, 0, 1, 1]
  stress:
    label: 压力骰
    faces:
      - { value: 0, text: 空白, weight: 4 }
      - { value: 1, text: 成功, weight: 3 }
      - { value: 2, text: 暴击, weight: 1 }
```

| 字段 | 作用 |
| --- | --- |
| `label` | 展示名。 |
| `faces` | 2-100 个骰面。数字可直接写；对象骰面写 `value`、可选 `text`、可选正整数 `weight`。 |

在骰点表达式中用 `dice('fate', 4)`，而不是发明新的模糊 `dF` 语法：

```yaml
rolls:
  check: "dice('fate', 4) + attr.skill"
```

`detail` 会保留骰面文本；`total` 是骰面的数值总和。`dice()` 的第一个参数必须是规则包内已定义的静态名称，次数必须为正整数且受系统最大骰子数限制。

### 9.2 `tables`

```yaml
tables:
  critical:
    label: 暴击效果
    entries:
      - { weight: 4, text: "伤害 +1" }
      - { weight: 1, text: "伤害翻倍" }
```

随机表字段：

| 字段 | 作用 |
| --- | --- |
| `label` | 展示名。 |
| `entries` | 1-100 条表项。 |
| `entries[].weight` | 正整数权重，默认 1。 |
| `entries[].text` | 用户可见文本，最长 300 字。 |
| `entries[].value` | 可选数值，可用于公式。 |
| `entries[].tags` | 可选结构化标签。 |

在命令中声明抽表，保证一次调用只抽一次：

```yaml
draws:
  critical: critical
output: "暴击效果：{table.critical.text}"
```

`draws` 会在 `rolls` 之后、`let` 之前执行。表项的 `value` 通过 `table.critical.value` 引用。

### 9.3 资源字段

HP、SAN、法术位、弹药等都只是带范围的普通数值字段：

```yaml
hp:     { type: integer, default: 10, min: 0, max: 10 }
mana:   { type: integer, default: 6, min: 0, max: 6 }
ammo:   { type: integer, default: 30, min: 0, max: 30 }
```

命令可通过 `add`、`subtract`、`clamp` 更新资源。运行时 2.0 还允许 `abilities` 声明角色或群资源字段和固定消耗；资源不足或能力仍在冷却时，整次使用失败且不会扣除资源。

## 10. 启用范围、权限、版本和回滚

### 10.1 作用域

- 导入的规则包由主人全局管理；
- 每个群单独选择启用/禁用哪个包和哪个版本；
- 自定义规则包当前只支持按群启用，私聊不会调用规则包；
- 人物卡数据按“QQ 用户 + 人物卡 + 规则包 ID”保存，不按群复制；
- 显示名称和执行结果仍只在当前消息所在群可见。

后续可以增加“群专属规则包”，但它必须使用独立命名空间，例如 `g-123456-my-rule`，不能与全局包同名覆盖。

### 10.2 版本

每次导入同一 `id` 都生成不可变版本。源 YAML 和规范化规则都会记录 SHA-256，确认、加载和导出时均会复验；磁盘内容被修改后该版本会拒绝使用。新版本不会自动替换任何群的活动版本。

```text
.骰规则导入                 # 得到 fate-lite@4 的预检
.骰规则确认 fate-lite       # 保存版本 4
.骰规则启用 fate-lite@4     # 当前群切换到版本 4
.骰规则回滚 fate-lite 3     # 当前群切回版本 3
```

导出 `<id>` 默认选择最新版本，指定旧版本时使用 `<id>@<版本>`。`.骰规则删除 <id> 确认` 会从所有群禁用该包并把规则文件移到 `archived`，人物卡数据始终保留；`.骰规则恢复 <id>` 会恢复最新有效归档，但不会重新启用任何群。归档和恢复对目录移动与索引提交做失败补偿。当前不提供批量彻底删除人物卡数据的命令。

### 10.3 命令冲突

以下命令属于系统保留命令，规则包不能覆写：

```text
.r .roll .ra .rc .rb .rp .rh .rah .sc .en .st .pc .nn
.set .setcoc .骰娘 .dice .骰规则 .log
```

即使规则包有 `aliases: [ra]`，导入也会失败。包 ID 与所有别名组成调用前缀；它们在当前群所有已启用规则包之间必须唯一。发生冲突时该包不会启用，需要修改 YAML 别名或先禁用冲突包。

## 11. 安全上限与禁止能力

系统硬上限不是建议，是拒绝导入或执行的边界：

| 项目 | 上限 |
| --- | --- |
| YAML 文件大小 | 64 KiB |
| YAML alias 展开 | 最多 50 次 |
| 规则包命令数 | 30 |
| 每条命令参数数 | 12 |
| 每条命令命名骰点数 | 12 |
| 单个公式长度 | 512 字符 |
| 公式 AST 深度 | 32 |
| 公式 AST 节点数 | 256 |
| 派生字段数 | 80 |
| 派生字段依赖深度 | 24 |
| 非标准骰骰面数 | 100 |
| 随机表条目数 | 100 |
| 单次命令骰子总数 | 继承系统 `maxDiceCount`，且规则包可进一步降低 |
| 单条输出长度 | 2,000 字符 |

规则包不能：

- 执行 JavaScript、`eval`、导入模块或访问文件；
- 请求网络、调用模型、调用机器人工具或读取群文件；
- 读取/修改未通过 `actor` 参数声明、目标类型校验和权限检查的其他人物卡；
- 修改 QQ 群名片、禁言、撤回、任意私聊或 @ 成员；`visibility` 只能向发起者或已识别的 GM/管理员交付本次结果；
- 无限递归、循环、随机重掷到满足条件为止；
- 覆盖核心骰娘命令。

这些限制是为了让“别人发来一个 YAML”仍然是可以安全预览和导入的行为。

### 12.1 Fate Lite：成功等级

```yaml
version: 1
id: fate-lite
name: Fate Lite
aliases: [fate, 命运]
character:
  fields:
    name: { type: string, default: "{sender.card}" }
    skill: { type: integer, default: 0, min: -3, max: 8 }
dice_sets:
  fate:
    label: Fate 骰
    faces: [-1, -1, 0, 0, 1, 1]
commands:
  - id: check
    aliases: [check, 检定]
    arguments:
      - { id: difficulty, type: integer, required: true }
    rolls:
      fate: "dice('fate', 4)"
    let:
      total: "roll.fate.total + attr.skill"
    branches:
      - when: "let.total >= arg.difficulty + 3"
        result: 出色成功
      - when: "let.total >= arg.difficulty"
        result: 成功
      - result: 失败
    output: "{actor}：{roll.fate.detail} + 技能 {attr.skill} = {let.total}，{result}。"
```

### 12.2 有状态伤害：不要把派生值当成可写值

若 `effective_strength` 是公式字段，就用它来判断；真正扣除的是 `hp`，不要对派生值 `set`：

```yaml
branches:
  - when: "roll.attack.total >= arg.defense"
    result: 命中
    actions:
      - op: subtract
        field: hp
        value: "roll.damage.total"
      - op: clamp
        field: hp
        min: 0
        max: "attr.max_hp"
```

### 12.3 角色名随角色数据变化

```yaml
identity:
  display_name: "{attr.name} - {attr.title}"
  fallback: "{sender.card}"
```

当用户执行 `.shadow 设 name=艾琳 title=夜行者` 后，以后的骰娘输出会自动显示“艾琳 - 夜行者”。这不是改 QQ 名片，也不会影响其他规则包的名称。

## 13. 校验、排错与兼容策略

### 13.1 导入前检查

导入必须全部通过才会写入规则库。预检包含：

- YAML 语法与文件大小；
- 顶层和所有嵌套字段 schema；
- ID、别名和保留命令；当前群前缀冲突在启用时检查；
- 字段引用是否存在，以及表达式是否只使用允许的阶段数据；
- 派生字段循环依赖；
- 骰点、随机表、模板、动作和上限；
- 至少一个无副作用示例试算。

失败不会改变已启用版本，也不会写入人物卡。

### 13.2 常见错误

| 提示 | 原因 | 修复 |
| --- | --- | --- |
| `commands[0].rolls.check: unknown field attr.aglity` | 属性拼写不存在。 | 改为正确的 `attr.agility`。 |
| `character.fields.a: cyclic dependency a -> b -> a` | 派生字段相互引用。 | 让其中一个变成可持久字段或删除循环。 |
| `actions[0]: field effective_strength is derived` | 试图写入 `formula` 字段。 | 改写源字段，例如 `injury`。 |
| `aliases contains reserved command ra` | 与内置命令冲突。 | 改为 `myra` 或使用规则包前缀。 |
| `division by zero` | 当前数据导致公式除零。 | 加条件或 `max(1, value)` 保护。 |
| `value 13 exceeds max 10 for hp` | 写入超出字段范围。 | 调整动作或增加 `clamp`。 |

### 13.3 未来兼容原则

- 格式 V1 已存在字段只追加，不改语义；废弃字段会至少保留一个大版本并给出迁移工具。
- 未知字段是错误而不是忽略，避免作者以为新能力已生效。
- 新能力优先以顶层可选区块添加，而不是改变现有 `commands` 的含义。2.0 已按此方式增加 `group`、`statuses`、`items`、`abilities` 和 `events`。
- 地图、异步选择、复杂区域效果等尚未实现的能力不会偷偷塞进普通表达式；未来仍应使用新的、可校验的声明式模块。
- 示例 YAML 是自动化回归夹具。规则运行时升级后，三份 V1 示例和 V2 团务示例都必须仍能导入、预览和执行。

## 14. 角色目标与权限

### 14.1 `actor` 参数

跨角色命令必须把目标声明成 `actor` 参数。下面的命令只允许选择群成员或本群 NPC，不允许省略目标：

```yaml
commands:
  - id: attack
    aliases: [attack, 攻击]
    arguments:
      - id: victim
        label: 目标
        type: actor
        required: true
        allowed: [member, npc]
      - id: damage
        type: integer
        required: true
        min: 0
        max: 20
    actions:
      - op: subtract
        scope: target
        target: victim
        field: hp
        value: "arg.damage"
    output: "{actor}攻击{target.name}，目标HP={target.attr.hp}。"
```

目标选择器是固定语法：

| 写法 | 含义 | 备注 |
| --- | --- | --- |
| `self`、`me`、`我`、`自己` | 命令发起者 | 参数的 `allowed` 必须含 `self`。 |
| `[CQ:at,qq=123456]` | 被 @ 的群成员 | QQ 事件中的标准 CQ at。 |
| `<@123456>` | 兼容 mention 形式 | 仍会按群成员处理。 |
| `qq:123456` | 指定 QQ | 推荐用于脚本化固定命令。 |
| `123456` | 纯 QQ | 只接受 5-20 位数字。 |
| `reply`、`回复` | 被引用消息的发言者 | 当前平台事件必须能读取引用。 |
| `npc:goblin` | 指定本规则包、本群的 NPC | NPC 必须先由 GM 创建。 |
| 精确群名片或昵称 | 精确匹配一个群成员 | 重名时报错，需改用 @ 或 QQ；包含空格时加引号。 |

找不到目标就是执行失败，不会把未识别文字当作当前用户，也不会猜一个相似昵称。

多目标参数：

```yaml
arguments:
  - id: victims
    type: actor
    required: true
    allowed: [member, npc]
    multiple: true
    rest: true
actions:
  - { op: subtract, scope: target, target: victims, field: hp, value: 1 }
```

此时固定命令形如 `.规则 群伤 qq:123456 npc:goblin`。每个剩余 token 都必须是一个合法目标，重复目标会去重。`multiple: true` 的 actor 参数必须放在最后并同时设置 `rest: true`。

### 14.2 权限等级

权限从低到高为：

| 等级 | 来源 | 能力 |
| --- | --- | --- |
| `player` | 默认群成员 | 执行玩家命令、查看和修改自己的非秘密字段、使用自己的已学能力。 |
| `gm` | 当前规则包、本群显式分配 | 管理 NPC、先攻、状态、物品、能力、团务和审计；可访问秘密字段。 |
| `admin` | QQ 群主或管理员自动获得 | 包含 GM 权限，并可分配/撤销本规则包 GM。 |
| `master` | 机器人主人 | 最高运行权限；同时负责全局规则包导入、确认、归档和恢复。 |

命令通过 `permission` 声明最低等级：

```yaml
commands:
  - id: keeper_note
    aliases: [keepernote, 主持笔记]
    permission: gm
    output: "秘密：{attr.secret_note}"
```

`secret` 字段、依赖秘密字段的派生字段、秘密资源能力和写秘密字段的 action，都要求命令至少为 `permission: gm`。导入时会静态拒绝明显的玩家命令泄露，运行时的通用卡命令还会再次鉴权。

固定权限命令：

```text
.规则 权限 列表
.规则 权限 设置 gm qq:123456
.规则 权限 设置 player qq:123456
```

只有 `admin` 或 `master` 能设置规则角色。设置 `player` 会删除该成员的显式 GM 身份；不会降低 QQ 群主/管理员自带的 `admin` 权限。

### 14.3 数据作用域

- 玩家人物卡：按“QQ + 当前 `.pc` 人物卡 + 规则包 ID”保存，可在启用同一规则包的不同群复用。
- NPC、规则角色、群字段、团务、先攻和审计：按“群 + 规则包 ID”保存，不跨群共享。
- 跨角色 action 只加载 YAML 中已声明的 actor 参数目标。
- 一条命令涉及的所有人物卡与群状态在同一状态事务中提交；任何参数、权限、表达式、范围、私密接收者或输出校验失败都不提交。

## 15. 私密结果、对抗与审计

### 15.1 `visibility`

| 值 | 私密正文接收者 | 群内内容 |
| --- | --- | --- |
| `public` | 无 | 发送 `output`。 |
| `private` | 命令发起者 | 发送 `public_output` 或默认暗骰占位。 |
| `gm` | 显式 GM、QQ 群主和群管理员 | 发送 `public_output` 或默认暗骰占位。 |

示例：

```yaml
commands:
  - id: secret_search
    aliases: [secret, 暗查]
    visibility: gm
    rolls:
      clue: "1d100"
    output: "秘密结果：{roll.clue.detail}"
    public_output: "{actor}进行了一次秘密调查，结果已交给GM。"
```

`visibility` 控制的是本次结果，不是任意消息能力：规则包不能指定任意 QQ，不能把 CQ 码塞进输出。`gm` 找不到任何合格接收者时，命令在状态提交前失败。已经找到接收者但平台私聊发送失败时，正文不会降级到群内；群内会列出未送达的 QQ，结构化审计仍保留本次已提交操作。

### 15.2 `opposed`

```yaml
opposed:
  target: rival
  actor_value: "1d20 + attr.power"
  target_value: "1d20 + target.attr.power"
  mode: higher
  tie: tie
output: "{opposed.actor} vs {opposed.target}，胜者={opposed.winner}"
```

| 字段 | 作用 |
| --- | --- |
| `target` | 引用本命令的单个 actor 参数 ID；多目标参数不能用于对抗。 |
| `actor_value` | 发起者一方数值表达式，可掷骰。 |
| `target_value` | 目标一方数值表达式，可引用 `target.*` 并掷骰。 |
| `mode` | `higher` 表示高者胜，`lower` 表示低者胜；默认 `higher`。 |
| `tie` | 相等时返回 `tie`、`actor` 或 `target`；默认 `tie`。 |

对抗上下文包含 `opposed.actor`、`opposed.target`、`opposed.result`、`opposed.winner` 和绝对差值 `opposed.margin`。双方骰点使用本次调用的同一确定性随机源，并计入全局骰数上限。

### 15.3 结构化审计与普通团录

每次自定义规则执行和团务固定命令都会生成审计 ID。审计记录最多保留最近 200 条，包含：

- 时间、规则包、包版本、命令、执行者和目标；
- 本次调用随机种子；
- 解析后的参数、骰点明细、抽表、对抗、结果标签；
- 每个 action 的目标、字段、变更前和值变更后；
- 可见性和完整规则输出。

固定查询：

```text
.规则 审计
.规则 审计 10
```

只有 GM 以上可查，一次最多展示 20 条摘要。`visibility: private/gm` 的完整正文不会写入可由普通团录导出的公开 content；普通 log 只保存群内占位和审计 ID。完整审计保存在规则群状态中。

## 16. 群状态、NPC、状态、物品与能力

### 16.1 `group.fields`

`group.fields` 的字段参数与 `character.fields` 相同，引用前缀改为 `shared`：

```yaml
group:
  fields:
    threat: { type: integer, default: 0, min: 0, max: 20 }
    gm_goal: { type: string, default: 未设置, secret: true }
```

普通玩家可以查看非秘密群字段；只有 GM 以上能设置群字段。

```text
.规则 群卡
.规则 群查 threat
.规则 群设 threat=3
```

YAML action 写群字段必须显式使用 `scope: group`。群 scope 不支持状态、物品或能力，因为这些属于具体角色。

### 16.2 NPC

NPC 的人物卡结构与玩家使用同一套 `character.fields`、状态、物品和能力定义，但数据只属于当前群和当前规则包。

```text
.规则 npc 列表
.规则 npc 创建 goblin 哥布林斥候
.规则 npc 卡 goblin
.规则 npc 设 goblin hp=8 name="灰牙"
.规则 npc 查 goblin hp
.规则 npc 删除 goblin
```

NPC ID 必须以小写字母开头，至少 2 个字符，只含小写字母、数字、下划线或连字符。创建、删除和修改均要求 GM；删除 NPC 会同时从先攻列表移除它，但不会改动任何玩家人物卡。

### 16.3 `statuses`

```yaml
statuses:
  poison:
    label: 中毒
    description: 行动结束时按层数扣血。
    default_duration: 2
    max_stacks: 3
    tick: turn_end
    on_apply: []
    on_tick:
      - { op: subtract, field: hp, value: "status.stacks" }
    on_expire: []
```

| 字段 | 作用 |
| --- | --- |
| `label` / `description` | 展示名和说明。 |
| `default_duration` | 默认持续次数；省略表示永久。范围 1-100000。 |
| `max_stacks` | 最大层数，默认 1，范围 1-1000。 |
| `tick` | `manual`、`turn_start`、`turn_end`、`round_start`、`round_end`。默认 `manual`。 |
| `on_apply` | 每次添加状态后执行。 |
| `on_tick` | 对应 tick 到来时执行，然后持续时间减 1。 |
| `on_expire` | 持续时间归零或主动移除时执行。 |

生命周期动作可读取 `status.id`、`status.stacks` 和 `status.duration`。递归动作深度最多 8；所有变更仍在当前回合事务内提交。

```text
.规则 状态 列表 [目标]
.规则 状态 添加 <目标> <状态ID> [持续] [层数]
.规则 状态 删除 <目标> <状态ID>
.规则 状态 结算 <目标>
```

列表可由玩家查看；添加、删除和手动结算要求 GM。`目标` 支持 self、@、QQ、reply 和 `npc:id`。

### 16.4 `items` 与装备

```yaml
items:
  potion:
    label: 治疗药水
    stackable: true
    max_quantity: 10
  shield:
    label: 盾牌
    stackable: false
    max_quantity: 1
    slot: hand
```

| 字段 | 作用 |
| --- | --- |
| `label` / `description` | 展示名和说明。 |
| `stackable` | 是否可堆叠；`false` 时 `max_quantity` 只能是 1。 |
| `max_quantity` | 数量上限，默认 1000000。 |
| `slot` | 可选装备槽；装备新物品时自动卸下同槽旧物品。 |

表达式可读取 `inventory.<物品ID>.quantity` 和 `.equipped`。固定命令：

```text
.规则 物品 列表 [目标]
.规则 物品 添加 <目标> <物品ID> [数量]
.规则 物品 删除 <目标> <物品ID> [数量]
.规则 物品 装备 <目标> <物品ID>
.规则 物品 卸下 <目标> <物品ID>
```

列表可由玩家查看；数量和装备变更要求 GM。数量不足、超过上限或装备不存在都会使整次操作失败。

### 16.5 `abilities`：技能、法术、冷却和资源

```yaml
abilities:
  fireball:
    label: 火球术
    kind: spell
    max_rank: 3
    cooldown: 2
    resource_scope: actor
    resource_field: mp
    resource_cost: 2
  rally:
    label: 集结
    kind: skill
    max_rank: 1
    cooldown: 1
    resource_scope: group
    resource_field: supplies
    resource_cost: 1
```

| 字段 | 作用 |
| --- | --- |
| `kind` | `skill` 或 `spell`，默认按技能展示。 |
| `max_rank` | 最高等级，默认 1。 |
| `cooldown` | 成功使用后写入的回合冷却，默认 0。 |
| `resource_scope` | `actor` 或 `group`，默认 `actor`。 |
| `resource_field` | 对应作用域中的可持久数值字段。 |
| `resource_cost` | 每次成功使用的固定消耗；设置后必须同时设置资源字段。 |

```text
.规则 技能 列表 [目标]
.规则 技能 学习 <目标> <能力ID> [等级]
.规则 技能 遗忘 <目标> <能力ID>
.规则 技能 使用 <能力ID>
.规则 技能 使用 <目标> <能力ID>
.规则 技能 重置 <目标> <能力ID>
```

玩家可使用自己的已学能力；学习、遗忘、替别人使用和重置要求 GM。使用时先检查是否已学习、是否处于冷却，再检查资源；失败不会扣资源。角色轮到 `turn_start` 时，该角色所有大于 0 的能力冷却减 1。秘密资源字段只能由 GM 使用。

## 17. 团务、先攻与生命周期事件

### 17.1 团务会话

```text
.规则 团务 状态
.规则 团务 开始 [标题]
.规则 团务 结束
```

查看状态对所有人开放；开始和结束要求 GM。开始时会初始化标题、轮次、回合、阶段和当前行动者，执行 `session_start`，并在当前群开启现有 `.log`。结束时执行 `session_end`、标记结束时间、清空当前行动者并停止 log。团录导出仍使用 `.log export`。

### 17.2 先攻与回合推进

```text
.规则 先攻 列表
.规则 先攻 添加 <目标> <数值>
.规则 先攻 删除 <目标>
.规则 先攻 开始
.规则 先攻 下一回合
.规则 先攻 结束
.规则 先攻 清空
```

列表对所有人开放，其余操作要求 GM。开始时按先攻值从高到低排序，同值按显示名排序；设为第 1 轮、第 1 回合，并让第一名进入 `turn_start`。`下一回合` 的顺序是：

1. 当前角色执行 `turn_end` 事件和 `turn_end` 状态 tick；
2. 如果行动表走完，执行 `round_end` 事件与全体 `round_end` 状态 tick，轮次加 1；
3. 新一轮执行 `round_start` 事件与全体 `round_start` 状态 tick；
4. 下一角色执行 `turn_start` 事件、冷却减 1 和 `turn_start` 状态 tick；
5. 所有角色和群字段改动一次提交并写入审计。

`先攻 结束` 只结束当前战斗推进并保留列表；`先攻 清空` 会清空列表、当前角色和轮次计数。只启动先攻不会自动创建普通 log，正式团务应先执行 `团务 开始`。

### 17.3 `events`

支持的事件名固定为：

| 事件 | 触发时机 |
| --- | --- |
| `session_start` | 团务开始。 |
| `session_end` | 团务结束。 |
| `round_start` | 第一轮开始和后续新轮开始。 |
| `round_end` | 一轮最后一个行动者结束后。 |
| `turn_start` | 每个行动者开始行动。 |
| `turn_end` | 当前行动者结束行动。 |

事件值是普通 action 数组，可使用 `actor` 或 `group` scope；事件不接受任意函数或脚本：

```yaml
events:
  session_start:
    - { op: set, scope: group, field: threat, value: 1 }
  round_start:
    - { op: add, scope: group, field: threat, value: 1 }
  session_end:
    - { op: set, scope: group, field: threat, value: 0 }
```

生命周期表达式可读取 `attr`、`derived`、`shared`、`session`、`inventory`、`abilities`、`status` 和当前事件角色。`session` 只开放 `active`、`title`、`round`、`turn`、`phase`。

## 18. 全部固定命令速查

以下 `<规则>` 表示当前群已启用包的 `id` 或 `aliases`。`.` 均可替换为中文句号 `。`。除 YAML 自己声明的子命令外，运行时内置命令为：

```text
.<规则>                                  查看该包命令帮助
.<规则> 卡 [目标]
.<规则> 设 [目标] field=value [...]
.<规则> 查 [目标] field
.<规则> 删 [目标] field

.<规则> 权限 列表
.<规则> 权限 设置 gm|player <成员>

.<规则> npc 列表
.<规则> npc 创建 <id> <名称>
.<规则> npc 删除 <id>
.<规则> npc 卡 <id>
.<规则> npc 设 <id> field=value [...]
.<规则> npc 查 <id> field

.<规则> 群卡
.<规则> 群设 field=value [...]
.<规则> 群查 field

.<规则> 团务 状态
.<规则> 团务 开始 [标题]
.<规则> 团务 结束

.<规则> 先攻 列表
.<规则> 先攻 添加 <目标> <数值>
.<规则> 先攻 删除 <目标>
.<规则> 先攻 开始|下一回合|结束|清空

.<规则> 状态 列表 [目标]
.<规则> 状态 添加 <目标> <状态ID> [持续] [层数]
```
